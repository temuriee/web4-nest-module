### 🧱 წარმოიდგინე ასეთი სცენარი

შენ გაქვს **ფორმა**, სადაც მომხმარებელი წერს თავის მონაცემებს (სახელი, ასაკი და ა.შ.)

👉 ეს მონაცემები მოდის სერვერში.

---

## 📦 DTO (Data Transfer Object) — „წესი / ფორმის ნიმუში“

DTO არის უბრალოდ **წესი**, რომელიც ამბობს:

> „მონაცემები უნდა იყოს ასეთი ფორმის“

მაგალითად:

- name → უნდა იყოს ტექსტი
- age → უნდა იყოს რიცხვი

ანუ DTO = **როგორი უნდა იყოს მონაცემი**

```ts
export class CreateUserDto {
  name: string;
  age: number;
}
```

---

## 🚰 Pipe — „შემოწმების ფილტრი“

Pipe არის რაღაც, რაც:

1. ამოწმებს მონაცემს
2. (საჭიროების შემთხვევაში) ასწორებს/გადაკეთებს

👉 ანუ Pipe = **ამოწმებს DTO-ს წესებს რეალურ მონაცემზე**

---

## 🤝 როგორ მუშაობენ ერთად?

აი ყველაზე მნიშვნელოვანი 👇

1. მომხმარებელი აგზავნის მონაცემს (მაგ: `{ name: "Gio", age: "20" }`)
2. DTO ამბობს:
   👉 „age უნდა იყოს number“
3. Pipe მოდის და:
   - ამოწმებს
   - შეიძლება გადააკეთოს `"20"` → `20`

---

## 🧠 მოკლედ

- **DTO** → ამბობს _როგორი უნდა იყოს მონაცემი_
- **Pipe** → ამოწმებს _მართლა ასეთია თუ არა_

---

## 🔥 რეალური მაგალითი NestJS-ში

```ts
@Post()
createUser(@Body() dto: CreateUserDto) {
  return dto;
}
```

თუ ჩართავ ValidationPipe-ს:

```ts
app.useGlobalPipes(new ValidationPipe());
```

👉 მაშინ:

- DTO + decorators (მაგ: `@IsString()`, `@IsNumber()`)
- Pipe ავტომატურად ამოწმებს ყველაფერს

---

## 🧩 კიდევ უფრო მარტივად

იფიქრე ასე:

- DTO = „კითხვარის შაბლონი“
- Pipe = „მასწავლებელი, რომელიც ამოწმებს სწორად შეავსე თუ არა“
