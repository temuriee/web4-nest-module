### 1️⃣ What this code does

```js
const jwt = require('jsonwebtoken');
```

- This imports a library that helps us make **JWTs (JSON Web Tokens)**.
- Think of JWT like a **secret badge** that proves: “Yes, this user is logged in!”

---

```js
const generateToken = (id) => {
  return jwt.sign({ id }, process.env.JWT, {
    expiresIn: '10d',
  });
};
```

Step by step:

1. **`{ id }`** → the info we put inside the token.
   - Usually the user’s ID.
   - Example: `{ id: "123" }` → “this token belongs to user 123”.

2. **`process.env.JWT`** → the **secret key**.
   - This is like the **lock** that only your server knows.
   - Only the server can **create** or **verify** the token using this secret.

3. **`expiresIn: "10d"`** → how long the token is valid.
   - `"10d"` = 10 days. After that, the token **won’t work**.

---

### 2️⃣ How it works in practice

- User logs in → server gives them a JWT.
- Every time user makes a request, they **send the JWT**.
- Server checks JWT → if valid → user is allowed. ✅

It’s like:

```text
User: "Here’s my secret badge!"
Server: "Cool, you’re allowed to enter."
```

---

### 3️⃣ Why `process.env.JWT` is important

- This is the **secret key**, like a master password to lock/unlock tokens.
- **It should NOT be random every time**, otherwise the server couldn’t verify old tokens.
- Think of it like a **stamp**:
  - Server stamps the badge with the secret → later server can check the stamp.
  - If the stamp keeps changing randomly → server can’t verify badges.

So yes, the secret is **same for all tokens** on your server, but the token itself is **unique** because it includes user ID and timestamp.

---

### 4️⃣ Token uniqueness

- Every token is **different**, even for the same user, because it includes:
  - The user ID
  - Timestamp (when it was created)
  - Expiry

- But **verification works** because the server uses the **same secret**.

---

✅ **Summary**

1. JWT = a secret badge for the user.
2. Server signs the badge with a secret key (like a stamp).
3. Token includes user ID + expiry.
4. Secret key stays the same so server can **check the badge** later.
5. Token is unique because of ID + creation time, even though the secret is the same.
