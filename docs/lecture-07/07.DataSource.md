### 🔹 რა არის `DataSource`?

`DataSource` არის **TypeORM-ის მთავარი ობიექტი**, რომელიც წარმოადგენს **კავშირის (connection)** ინფორმაციას და საშუალებას გაძლევს **მოწვდე ბაზასთან პირდაპირ**, ანუ:

- შეიცავს ინფორმაციას **რომელ ბაზასთან ხარ დაკავშირებული**
- ინახავს **EntityManager-ს**, რომელიც გეხმარება CRUD ოპერაციებში
- იძლევა საშუალებას **გააკეთო custom query-ები**, transaction-ები და სხვა დაბალი დონის ოპერაციები

---

### 🔹 როგორ ხდება მისი ინექცია NestJS-ში?

NestJS-ში შეგიძლია “ჩასვა” (inject) `DataSource` კლასში პირდაპირ constructor-ით — ზუსტად ისე, როგორც repository ან service:

```ts
@Injectable()
export class PostsService {
  constructor(
    @InjectRepository(Post)
    private readonly postsRepository: Repository<Post>,

    // Inject datasource
    private readonly dataSource: DataSource,
  ) {}
}
```

ამით NestJS ავტომატურად მოიყვანს იმ `DataSource` instance-ს, რომელიც შექმნილია შენი TypeORM კონფიგურაციით (`TypeOrmModule.forRoot()`-ში).

---

### 🔹 რისთვის გამოიყენება პრაქტიკაში?

`DataSource` გაძლევს საშუალებას იმუშაო უფრო დაბალ დონეზე, ვიდრე `Repository`.

მაგალითად:

#### ✅ 1. Transaction (ტრანზაქცია):

თუ გინდა ერთზე მეტი query იმუშაოს ერთად (ან ყველაფერი შესრულდეს ან არაფერი):

```ts
await this.dataSource.transaction(async (manager) => {
  const post = await manager.save(Post, { title: 'New Post' });
  const meta = await manager.save(Meta, { post });
});
```

#### ✅ 2. Raw query (სუფთა SQL):

თუ გინდა პირდაპირ SQL გაუშვა:

```ts
const result = await this.dataSource.query(
  'SELECT * FROM posts WHERE id = $1',
  [1],
);
```

#### ✅ 3. Custom repository შექმნა:

შეგიძლია შენი საკუთარი repository ააგო `DataSource.getRepository()`-ით:

```ts
const postRepository = this.dataSource.getRepository(Post);
const posts = await postRepository.find();
```

### 🔹 მოკლედ:

| სახელი       | აღწერა                                                        |
| ------------ | ------------------------------------------------------------- |
| `DataSource` | TypeORM-ის “გლობალური” ობიექტი, რომელიც მართავს DB კავშირს    |
| შეიცავს      | EntityManager, Connection Options, QueryRunner                |
| გამოიყენება  | Transaction-ებში, custom query-ებში, custom repository-ებში   |
| ინექცია      | ავტომატურად ხდება NestJS-ის მიერ `TypeOrmModule`-ის მეშვეობით |
