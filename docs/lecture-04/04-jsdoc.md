## 🧩 მარტივად რომ ვთქვათ:

JSDoc არის **„ჭკვიანი კომენტარები“**, რომლებიც ეუბნება:

- ადამიანებს — რას აკეთებს კოდი
- IDE-ს (მაგ. VS Code) — რა ტიპის მნიშვნელობებია (გიჩვენებს подсказки/autocomplete)

---

## ✍️ სინტაქსი (გამოიყენება /\*_ ... _/ ფორმით):

მაგალითი:

```js
/**
 * ამ ფუნქციას აძლევ ორ რიცხვს და აბრუნებს მათ ჯამს.
 * @param {number} a - პირველი რიცხვი
 * @param {number} b - მეორე რიცხვი
 * @returns {number} a და b-ის ჯამი
 */
function add(a, b) {
  return a + b;
}
```

🟢 რაც მივიღეთ:

- `@param {type} name` — აღწერს პარამეტრის ტიპს და როლს
- `@returns {type}` — რა ტიპის მნიშვნელობას აბრუნებს ფუნქცია
- ტექსტი ზემოთ — აღწერა ადამიანის წასაკითხად

## ⚡ მაგარი ბონუსი:

TypeScript-ში ბევრი რამე უკვე ჩაშენებულია (ტიპები და კომენტარები),
მაგრამ JSDoc მაინც გამოიყენება როცა:

- წერ **სუფთა JavaScript-ში** (არა TypeScript)
- ან გინდა **დამატებითი აღწერა** (მაგ. მაგალითების ან usage-ის ჩასმა)

## 🔹 მარტივი მაგალითი (კლასზე):

```js
/**
 * მომხმარებლის კლასი
 * @class
 */
class User {
  /**
   * ქმნის ახალ მომხმარებელს
   * @param {string} name - მომხმარებლის სახელი
   * @param {number} age - ასაკი
   */
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
}
```

### 🧾 მოკლედ:

| ელემენტი      | რას აკეთებს                             |
| ------------- | --------------------------------------- |
| `@param`      | აღწერს ფუნქციის არგუმენტებს             |
| `@returns`    | რა აბრუნებს ფუნქცია                     |
| `@class`      | აღნიშნავს კლასს                         |
| `@typedef`    | ქმნის საკუთარ ტიპს                      |
| `@example`    | usage მაგალითი                          |
| `@deprecated` | აფრთხილებს, რომ ეს ფუნქცია მოძველებულია |
