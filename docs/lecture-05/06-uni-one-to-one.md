## 🎒 წარმოვიდგინოთ ერთი პატარა ისტორია:

ჩვენ გვაქვს ორი რამ:

- 🧍‍♂️ **მოსწავლე** (User)
- 🪪 **მოსწავლის პროფილი** (Profile)

მოსწავლის პროფილში წერია:

> სახელი, ასაკი, ფოტო, ტელეფონი, ქალაქი...

---

### ახლა, როგორ ვუკავშირებთ ამ ორს?

თუ ვთქვათ:

> “**თითო მოსწავლეს** აქვს **ერთი პროფილი**”

ეს არის **One To One Relationship** — ერთი ერთზე კავშირი.

---

მაგრამ ახლა მოდი განვიხილოთ სიტყვა **Uni-directional** (ერთმხრივი):

### 🧩 Uni-directional ნიშნავს — მხოლოდ ერთი მხარე “იცის” მეორის არსებობა

ანუ:

- `User` იცის, რომ მას ჰყავს `Profile`
- მაგრამ `Profile` **არ იცის**, ვის ეკუთვნის

---

### მაგალითი კოდით (უბრალო ენით):

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile) // ერთი პროფილი თითო იუზერზე
  @JoinColumn() // პროფილი შეინახე აქ
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  age: number;
}
```

🧠 აქ `User`–ს აქვს ინფორმაცია თავის `Profile`-ზე,
მაგრამ `Profile`–ს **არ აქვს არანაირი ველი**, რომ თქვას “მე ამ იუზერს ვეკუთვნი”.

---

### რეალურ ცხოვრებაში ანალოგია 🎯

წარმოიდგინე:

- მოსწავლე იცის თავისი პროფილი (“მე მქვია ლევანი, ჩემი პროფილი არის ეს!”)
- მაგრამ პროფილზე მხოლოდ ინფორმაცია წერია (სახელი, ასაკი...)
  → თვითონ პროფილმა **არ იცის**, რომ ეს ლევანისაა.

ესაა **Uni-directional One-to-One**
(ერთმხრივი ერთი ერთზე კავშირი)

---

თუ ორივეს გავუგზავნეთ ინფორმაცია ერთმანეთზე, ანუ:

- მოსწავლე იცის თავისი პროფილი
- პროფილიც იცის, რომ ლევანისია

მაშინ ეს იქნება 👉 **Bi-directional One-to-One relationship**

---

### მოკლე შეჯამება:

| ტიპი                           | როგორია კავშირი | რომელი იცის რომელი                    |
| ------------------------------ | --------------- | ------------------------------------- |
| **Uni-directional One-to-One** | ერთი ერთზე      | მხოლოდ ერთი მხარე იცის მეორის შესახებ |
| **Bi-directional One-to-One**  | ერთი ერთზე      | ორივე მხარე იცნობს ერთმანეთს          |
