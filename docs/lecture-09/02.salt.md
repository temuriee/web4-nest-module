### 🧂 რას ნიშნავს "salt"

**Salt** (ინგლისურად “მარილი”) —
ეს არის _შემთხვევითი დამატება პაროლზე_ **შეინახვამდე**, რომ მისი გატეხვა ბევრად უფრო ძნელი იყოს.

---

### 🧠 როგორ მუშაობს მარტივად

წარმოიდგინე, რომ გაქვს პაროლი:

```
mypassword
```

თუ ამას პირდაპირ დავაჰეშავ:

```
hash(mypassword) → "ab34gh56..."
```

ახლა, თუ ვიღაცას აქვს ეგ ჰეში,
ის შეიძლება სცადოს _ყველა შესაძლო პაროლი_ სანამ იპოვის იმას, რაც ემთხვევა —
ეს נקდება **brute force attack** 😈

---

### 🔐 ახლა დავამატოთ “მარილი”

სანამ ჰეშს გააკეთებ,
დაამატე შემთხვევითი სიტყვა, მაგალითად `"x7!d9"`
და მერე ერთად გააკეთე ჰეში:

```
hash("mypasswordx7!d9") → "9f6h2j..."
```

ეს `"x7!d9"` არის შენი **salt** 🧂

მაშინაც კი, თუ სხვა ადამიანს აქვს იგივე პაროლი `"mypassword"`,
მისი ჰეში სხვანაირი იქნება,
რადგან მას სხვა salt დაემატება.

---

### 📦 როგორ ინახება

ხშირად bcrypt ან სხვა ბიბლიოთეკები
**salt-ს ავტომატურად ამატებენ და ინახავენ ჰეშთან ერთად**, მაგალითად ასე:

```
$2b$10$Kflkfs82ks9dOdkfl3... (bcrypt ჰეში + salt)
```

ანუ შენს ბაზაში ინახება უკვე "დაშიფრული" პაროლი + salt ერთად.

---

### 💡 მარტივად რომ ვთქვათ:

| ტერმინი      | რა აკეთებს                                       |
| ------------ | ------------------------------------------------ |
| **Password** | შენი ნამდვილი პაროლი (“mypassword”)              |
| **Salt**     | შემთხვევითი “მარილი”, რომელიც ემატება პაროლს     |
| **Hash**     | საბოლოო დაშიფრული შედეგი, რომელიც ინახება ბაზაში |

---

### 🧩 რატომ გვჭირდება salt

თუ ორი ადამიანს აქვს ერთნაირი პაროლი,
მაგალითად ორივეს აქვს `"123456"`,
salt-ის გარეშე მათი ჰეში იქნებოდა **სწორად ერთი და იგივე** —
და ჰაკერს ადვილად მიხვდებოდა, რომ ორივეს ერთი პაროლი აქვს 😬

მაგრამ salt-ის გამო:

```
user1 → hash("123456" + salt1)
user2 → hash("123456" + salt2)
```

მიღებული შედეგი იქნება სრულიად განსხვავებული ✅
