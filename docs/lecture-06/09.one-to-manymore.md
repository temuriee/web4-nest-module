## 🎯 შენ აკეთებ ამას:

```ts
// 🧍‍♂️ User
@OneToMany(() => Post, (post) => post.author)
posts: Post[];

// 📝 Post
@ManyToOne(() => User, (user) => user.posts)
author: User;
```

ეს არის **bi-directional One-To-Many / Many-To-One relationship**
ანუ — ორივე მხარე “იცნობს” ერთმანეთს.

---

## ❓ მაშ, რატომ ჩანს `author` პოსტში, მაგრამ `posts` იუზერში — არა?

### იმიტომ რომ:

**Foreign Key** ყოველთვის იქ იწერება, სადაც არის `@ManyToOne`.

ანუ:

- **Post** ცხრილში შეიქმნება foreign key `authorId`
- **User** ცხრილში **არ შეიქმნება** არაფერი ახალი

---

### ამიტომაც:

- როცა შენ ქმნი ახალ პოსტს, ბაზა ინახავს მაგალითად ასე:

| id  | title | authorId |
| --- | ----- | -------- |
| 1   | Hello | 5        |

👉 აქედან გამოდის, რომ `Post` **იცის** თავისი `author` (User).

---

მაგრამ:

- `User` ცხრილში **არ აქვს სია პოსტების** — ბაზაში ეს ინფორმაცია პირდაპირ არ იწერება.
- TypeORM კი “ვირტუალურად” ააგებს ამ კავშირს, როცა მას ეტყვი, რომ გამოიტანს.

---

## 💡 ანუ “რატომ არ ჩანს user-ში პოსტები?”

როცა შენ ეძებ `User`-ს ჩვეულებრივად:

```ts
const user = await userRepository.findOne({ where: { id: 1 } });
```

👉 აქ TypeORM **არ მოიტანს** `posts`, იმიტომ რომ ეს არის **lazy field** (ის ვერ მოაქვს ავტომატურად).

---

## ✅ როგორ მოიტანო, რომ `posts` გამოჩნდეს:

შენ უნდა უთხრა TypeORM-ს რომ მოიტანოს ეს კავშირიც:

```ts
const user = await userRepository.findOne({
  where: { id: 1 },
  relations: ['posts'], // 👈 უთხარი, რომ პოსტებიც მოიტანოს
});
```

ახლა მიიღებ:

```js
{
  id: 1,
  name: "Mindia",
  posts: [
    { id: 1, title: "My first post" },
    { id: 2, title: "My second post" }
  ]
}
```

---

## ✍️ მაგრამ პოსტზე რატომ ჩანს author?

რადგან `Post` ცხრილში არის `authorId` ველი (foreign key),
TypeORM მარტივად პოულობს შესაბამის `User`-ს.

ანუ:

- Post იცის author ვინ არის (ბაზაში FK-ით)
- მაგრამ User არ იცის თავისი პოსტები მანამდე, სანამ არ ეტყვი “გააერთიანე” (`relations`).

---

## 🔍 მოკლე შეჯამება:

| მხარე  | დეკორატორი | სად იწერება foreign key | ავტომატურად ჩანს?                              |
| ------ | ---------- | ----------------------- | ---------------------------------------------- |
| `Post` | @ManyToOne | ✅ Post ცხრილში         | ✅ (author ჩანს)                               |
| `User` | @OneToMany | ❌ User ცხრილში არა     | ❌ (posts არ ჩანს, სანამ relations არ მოითხოვ) |

მოკლედ რომ გითხრა:
👉 `@JoinColumn()` **არ არის აუცილებელი**, მაგრამ როცა წერ, შეგიძლია აკონტროლო **როგორ და რა სახელით შეიქმნება foreign key**.

---

## 🎯 მოდი ვნახოთ ორივე შემთხვევა:

---

### ✅ ვარიანტი #1 – როცა **არ წერ `@JoinColumn()`**

```ts
@ManyToOne(() => User, (user) => user.posts)
author: User;
```

TypeORM თავისით შექმნის foreign key სვეტს, სახელის წესით:

```
<propertyName> + <referencedColumnName>
```

👉 ანუ შენს შემთხვევაში იქნება:

```bash
authorId
```

---

### ✅ ვარიანტი #2 – როცა წერ `@JoinColumn({ name: "author_id" })`

```ts
@ManyToOne(() => User, (user) => user.posts)
@JoinColumn({ name: "author_id" })
author: User;
```

ამ დროს **შენ თვითონ განსაზღვრავ**, როგორ ერქმევა foreign key სვეტს.

👉 ბაზაში შეიქმნება ასე:

```bash
author_id
```

---

## 🧱 ბაზაში განსხვავება:

| ვარიანტი                                | foreign key column name |
| --------------------------------------- | ----------------------- |
| ❌ `@JoinColumn()` არ წერია             | `authorId`              |
| ✅ `@JoinColumn({ name: "author_id" })` | `author_id`             |

---

## 📖 სხვაობა სიტყვიერად:

- თუ არ დაწერ `@JoinColumn()` → TypeORM **თვითონ შექმნის** FK სვეტის სახელს camelCase სტილში (`authorId`)
- თუ დაწერ `@JoinColumn()` → შეგიძლია **სახელი აკონტროლო** (`author_id`, `user_id`, `created_by`, რაც გინდა)

---

## ⚙️ კიდევ ერთი პატარა ტექნიკური მნიშვნელობა

`@JoinColumn()` ასევე:

- **ნიშნავს**, რომ ეს მხარეა “relationship-ის მფლობელი” (owner side)
  ანუ აქ უნდა იყოს foreign key.
- TypeORM-იც ამით ხვდება, რომ FK უნდა ჩაწეროს **ამ entity-ში**, და არა მეორე მხარეს.

მაგრამ `@ManyToOne` ისედაც ყოველთვის არის owner side, ამიტომაც ამ შემთხვევაში `@JoinColumn()` უბრალოდ _დამატებითი კონტროლია_, არა აუცილებელი.

---

## 🧠 მოკლე დასამახსოვრებელი ფრაზა:

> `@JoinColumn()` მხოლოდ სახელს ან დეტალებს აკონტროლებს,
> მაგრამ თუ არ დაწერ — TypeORM მაინც შექმნის foreign key-ს ავტომატურად.
