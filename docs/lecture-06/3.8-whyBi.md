მოკლედ რომ ვთქვათ:

> **Bi-directional ურთიერთობა გვეხმარება, რომ ორივე მხრიდან მარტივად მივწვდეთ კავშირს — და არა მხოლოდ ერთი მხრიდან.**

---

## 🧩 Uni-directional

მხოლოდ ერთი მხარე “იცის” მეორის შესახებ.

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile)
  @JoinColumn()
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  age: number;
}
```

🧠 აქ:

- `User` იცის თავისი `Profile`
- მაგრამ `Profile` საერთოდ **არ იცის**, ვის ეკუთვნის.

---

### 👉 შედეგი კოდში:

```ts
const user = await userRepo.findOne({
  where: { id: 1 },
  relations: ['profile'],
});

console.log(user.profile.age); // ✅ მუშაობს
```

მაგრამ ახლა პირიქით სცადე:

```ts
const profile = await profileRepo.findOne({ where: { id: 1 } });
console.log(profile.user.name); // ❌ error — user არ არსებობს!
```

---

## 🧩 Bi-directional

ორივე მხარე “იცნობს” ერთმანეთს.

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile, (profile) => profile.user)
  @JoinColumn()
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  age: number;

  @OneToOne(() => User, (user) => user.profile)
  user: User;
}
```

---

### 👉 ახლა შეგიძლია ორივე მხრიდან იმუშაო:

```ts
// იუზერიდან პროფილამდე
const user = await userRepo.findOne({
  where: { id: 1 },
  relations: ['profile'],
});
console.log(user.profile.age); // ✅ მუშაობს

// პროფილიდან იუზერამდე
const profile = await profileRepo.findOne({
  where: { id: 1 },
  relations: ['user'],
});
console.log(profile.user.name); // ✅ მუშაობს
```

---

## 🎯 ანუ რა მოგცემს Bi-directional?

### 1️⃣ **მარტივი წვდომა ორივე მხრიდან**

შეგიძლია:

- იპოვო იუზერი და ნახო მისი პროფილი ✅
- ან იპოვო პროფილი და ნახო მისი იუზერი ✅

Uni-directional შემთხვევაში მეორე ვერ გააკეთებდი.

---

### 2️⃣ **Queryების სიმარტივე**

როცა გინდა ინფორმაცია ორივე მხრიდან, bi-directional გაძლევს უფლებას გამოიყენო `relations` ორივე მხარეს:

```ts
await profileRepo.find({ relations: ['user'] });
await userRepo.find({ relations: ['profile'] });
```

Uni-directional მოდელში მხოლოდ ერთი მხარე იმუშავებს `relations`-ით.

---

### 3️⃣ **Cascade და Relation ოპერაციები**

თუ გაქვს `cascade: true`,
bi-directional კავშირი საშუალებას გაძლევს TypeORM-მა ორივე მხარე სწორად “მოაშველოს”.

მაგალითად, შეგიძლია ერთდროულად შეინახო ორივე:

```ts
const profile = new Profile();
profile.age = 18;

const user = new User();
user.profile = profile;

await userRepo.save(user); // ✅ ორივე შეიქმნება სწორად
```

მერე როცა ამოიღებ პროფილს:

```ts
const profile = await profileRepo.findOne({
  where: { id: 1 },
  relations: ['user'],
});
console.log(profile.user); // ✅ უკვე იუზერიც მოდის
```

---

### 4️⃣ **ბმების სინქრონიზაცია ორმხრივად**

Bi-directional გაძლევს საშუალებას, რომ ობიექტების სტრუქტურა ყოველთვის იყოს სინქრონში.

მაგალითად:

```ts
user.profile = profile;
profile.user = user;
```

ორივე მხარე იცის ერთმანეთზე — და ORM ზუსტად მიხვდება, როგორ დააკავშიროს ბაზაში.

Uni-directional შემთხვევაში ეს ვერ იმუშავებს — მხოლოდ ერთ მხარეს შეინახება ბმა.

---

### 5️⃣ **მკითხველობა და ლოგიკური გამჭვირვალობა**

თუ მოდელები ურთიერთდამოკიდებულია (მაგალითად, `Profile` ყოველთვის უნდა ეკუთვნოდეს კონკრეტულ `User`-ს),
ლოგიკურადაც ჯობს, რომ პროფილმაც “იცოდეს”, ვის ეკუთვნის.

ეს უფრო ბუნებრივია კოდში:

```ts
console.log(profile.user.name);
```

ვიდრე ყოველ ჯერზე მექანიკურად მოძებნო იუზერი `profileId`-ით.

## 🧠 მარტივი შედარება:

| ფუნქცია                | Uni-directional      | Bi-directional                            |
| ---------------------- | -------------------- | ----------------------------------------- |
| წვდომა მეორე მხარეზე   | მხოლოდ ერთი მხრიდან  | ორივე მხრიდან                             |
| `relations` მხარდაჭერა | მხოლოდ ერთ მხარეს    | ორივე მხარეს                              |
| `cascade` მუშაობა      | შეზღუდულად           | სრულად                                    |
| კოდში სიმარტივე        | მარტივი              | ოდნავ მეტი სტრუქტურა, მაგრამ უფრო მოქნილი |
| კარგი როცა...          | მარტივი მოდელი გაქვს | ორმხრივი ურთიერთობა გჭირდება              |
