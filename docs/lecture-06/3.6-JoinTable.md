## 🎒 რას აკეთებს `@JoinColumn()`

👉 მოკლედ:
`@JoinColumn()` ამბობს TypeORM–ს:

> “🧱 ამ მხარეზე უნდა შეიქმნას foreign key column.”

ანუ —
**სად** უნდა შეინახოს TypeORM ის “სხვისი ID”,
რომ ორი ცხრილი ერთმანეთს დაუკავშიროს.

---

## 🧩 მაგალითი 1 — Uni-directional One-to-One

### 🎯 იდეა:

- `User` იცის თავისი `Profile`
- `Profile` არაფერს იცის `User`–ზე

---

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile)
  @JoinColumn() // ✅ აქ შეინახე foreign key
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  age: number;
}
```

---

### ⚙️ რეალურად ბაზაში ხდება ასე:

| users | columns                      |
| ----- | ---------------------------- |
| id    | profileId (FK → profiles.id) |

| profiles | columns |
| -------- | ------- |
| id       | age     |

---

🧠 ანუ `User` ცხრილში დაემატება სვეტი `profileId`,
რადგან `@JoinColumn()` სწორედ ამ მხარეზეა დართული.

📘 თუ `@JoinColumn()` არ ჩააწერ, TypeORM ვერ გაიგებს სად უნდა შეინახოს foreign key,
და დაგიშვებს შეცდომას ან საერთოდ არ შექმნის FK-ს.

---

## 🧩 მაგალითი 2 — Bi-directional One-to-One

### 🎯 იდეა:

- `User` იცის თავისი `Profile`
- `Profile` იცის თავისი `User`

---

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile, (profile) => profile.user)
  @JoinColumn() // ✅ მხოლოდ ერთ მხარეს იწერება!
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  age: number;

  @OneToOne(() => User, (user) => user.profile)
  user: User;
}
```

---

### ⚙️ რეალურად ბაზაში ხდება ასე:

| users | columns                      |
| ----- | ---------------------------- |
| id    | profileId (FK → profiles.id) |

| profiles | columns |
| -------- | ------- |
| id       | age     |

---

🧠 აქ მნიშვნელოვანია:

- `@JoinColumn()` იწერება **მხოლოდ ერთ მხარეს** —
  იმ მხარეზე, რომელიც “ფლობს” კავშირს (the **owning side**).
- მეორე მხარეს (Profile) **არ უნდა** იყოს `@JoinColumn()` —
  იმიტომ რომ ის მხოლოდ “პირიქით” მხარეა (inverse side).

---

## 🚦 როგორ ვიგოთ, რომელი მხარეა "owning side"?

- **ვინც შეიცავს `@JoinColumn()`**,
  ისაა _owning side_ — ანუ სადაც foreign key ჩაიწერება.

- მეორე მხარე, რომელსაც აქვს `(user) => user.profile`,
  მხოლოდ აღწერს უკუღმა ბმას (არ ინახავს foreign key-ს).

---

## 🧠 შეჯამება ცხრილში

| ტიპი                           | რომელ მხარეს იწერება `@JoinColumn()`        | ახსნა                                                           |
| ------------------------------ | ------------------------------------------- | --------------------------------------------------------------- |
| **Uni-directional One-to-One** | იმ მხარეს, რომელიც ერთადერთია (მაგ. `User`) | რადგან მხოლოდ ერთი entity იცნობს მეორეს                         |
| **Bi-directional One-to-One**  | მხოლოდ ერთ მხარეს — "owning side"           | მეორე მხარე უბრალოდ უკავშირებს, მაგრამ არ ინახავს foreign key-ს |

---

## 💬 დასამახსოვრებელი ფრაზა

> “`@JoinColumn()` არის ადგილი, სადაც შენ **ინახავ უცხო გასაღებს (foreign key)**.
> თუ ორივე მხარე ინახავს — ბაზაში ორი FK გაჩნდება ❌
> ამიტომ ის უნდა იყოს **მხოლოდ ერთ მხარეზე**.”

## 🎯 ძირითადი პრინციპი:

> **`@JoinColumn()` ყოველთვის იწერება მხოლოდ ერთხელ**
> — იმ მხარეზე, რომელიც **ფლობს ურთიერთობას** (owning side).

---

## 🔹 Uni-directional One-to-One

მხოლოდ ერთი მხარე “იცის” მეორეზე → ის ავტომატურადაა “owner”.

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => Profile)
  @JoinColumn() // ✅ აქ, რადგან ეს მხარე იცის Profile-ზე
  profile: Profile;
}
```

🧠 აქ:

- `Profile` საერთოდ არაფერს იცის `User`-ზე
- ამიტომ `User` ხდება "owning side"
- და **`@JoinColumn()` იწერება აქ**

---

## 🔹 Bi-directional One-to-One

ორივე მხარე იცნობს ერთმანეთს, მაგრამ **ერთმა მაინც უნდა იყოს “მეპატრონე” (owner)**.

```ts
@Entity()
export class User {
  @PrimaryGeneratedColumn()
  id: number;

  // პირველ არგუმენტში ვწერთ მეორე Entity-ს
  // მეორეში კი ვეუბნებით — მეორე მხარეს რომელი ველი უკავშირდება
  @OneToOne(() => Profile, (profile) => profile.user)
  @JoinColumn() // ✅ მხოლოდ აქ
  profile: Profile;
}

@Entity()
export class Profile {
  @PrimaryGeneratedColumn()
  id: number;

  @OneToOne(() => User, (user) => user.profile)
  user: User;
}
```

🧠 აქ:

- `User` არის **owning side** → foreign key შეინახება მის ცხრილში.
- `Profile` არის **inverse side** → მხოლოდ უკავშირებს, მაგრამ არაფერი ინახება.

---

## ⚙️ სხვაობა Uni ↔ Bi შორის სინტაქსში

| ელემენტი                  | Uni-directional                       | Bi-directional                                        |
| ------------------------- | ------------------------------------- | ----------------------------------------------------- |
| `@OneToOne()`             | `@OneToOne(() => Profile)`            | `@OneToOne(() => Profile, (profile) => profile.user)` |
| `@JoinColumn()`           | საჭიროა                               | საჭიროა მხოლოდ ერთ მხარეს                             |
| მეორე entity-ს ველი       | არ არის                               | აუცილებლად უნდა არსებობდეს (მაგ. `user`)              |
| ვინ ინახავს foreign key-ს | ის მხარე, სადაც `@JoinColumn()` წერია | ის მხარე, სადაც `@JoinColumn()` წერია                 |
