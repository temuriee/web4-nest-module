## 🎒 პატარა ისტორია #2

ჩვენ გვაქვს ორი რამ:

- 🧑‍🏫 **მასწავლებელი (Teacher)**
- 🧑‍🎓 **მოსწავლე (Student)**

და ვიცით, რომ:

> ერთ მასწავლებელს შეიძლება ჰყავდეს ბევრი მოსწავლე
> მაგრამ თითო მოსწავლეს ჰყავს მხოლოდ ერთი მასწავლებელი

ეს არის 👉
✅ **One-To-Many (ერთი–მრავალზე)**
და ამავე დროს
✅ **Many-To-One (მრავალი–ერთზე)**

---

## 💡 მარტივი სიტყვებით:

- **Teacher → Students** 👉 ერთი მასწავლებელი → ბევრი მოსწავლე
- **Student → Teacher** 👉 ბევრი მოსწავლე → ერთი მასწავლებელი

---

## 🎯 როგორ ვწერთ კოდში TypeORM-ში (TypeScript სინტაქსით)

### 🧑‍🏫 `Teacher` entity (ერთი მასწავლებელი ბევრ მოსწავლესთან)

```ts
@Entity()
export class Teacher {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  // 🧩 ერთი მასწავლებელი ბევრ მოსწავლესთან
  @OneToMany(() => Student, (student) => student.teacher)
  students: Student[];
}
```

---

### 🧑‍🎓 `Student` entity (მრავალი მოსწავლე ერთ მასწავლებელთან)

```ts
@Entity()
export class Student {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  // 🪪 ეს არის foreign key — მოსწავლის “მასწავლებლის ID”
  @ManyToOne(() => Teacher, (teacher) => teacher.students)
  @JoinColumn({ name: 'teacher_id' }) // optional, სახელის მითითება
  teacher: Teacher;
}
```

## 🧠 როგორ უნდა წარმოვიდგინო:

- მასწავლებელი **იცის თავისი ყველა მოსწავლე** (OneToMany)
- მოსწავლე **იცის თავისი მასწავლებელი** (ManyToOne)

ეს უკვე არის **Bi-directional** კავშირი (ორივე მხარე “იცნობს” ერთმანეთს).

---

## 🔄 Uni-directional ვარიანტიც შეგვიძლია

თუ გვინდა, რომ მხოლოდ **Student** იცნობდეს **Teacher**-ს
და **Teacher**-ს საერთოდ არ აინტერესებდეს ვინ ჰყავს, ვწერთ მხოლოდ ამას 👇

```ts
@Entity()
export class Student {
  @PrimaryGeneratedColumn()
  id: number;

  @Column()
  name: string;

  @ManyToOne(() => Teacher)
  @JoinColumn({ name: 'teacher_id' })
  teacher: Teacher;
}
```

🧩 ეს არის **Uni-directional Many-To-One**
(მოსწავლე იცის მასწავლებელი, მაგრამ მასწავლებელი არ იცის მოსწავლეები)

---

## 🧠 მოკლე ცხრილი დასამახსოვრებლად:

| კავშირი                            | აღწერა       | ვინ იცის ვინზე       | მაგალითი            |
| ---------------------------------- | ------------ | -------------------- | ------------------- |
| **One-To-One (Uni)**               | ერთი ერთზე   | მხოლოდ ერთი მხარე    | User → Profile      |
| **One-To-One (Bi)**                | ერთი ერთზე   | ორივე მხარე          | User ↔ Profile     |
| **One-To-Many / Many-To-One (Bi)** | ერთი ბევრთან | ორივე მხარე          | Teacher ↔ Students |
| **Many-To-One (Uni)**              | ბევრი ერთთან | მხოლოდ “ბევრი” მხარე | Students → Teacher  |
