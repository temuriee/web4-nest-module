## 🎯 რა არის `@OneToOne(() => MetaOption, { cascade: true })`

წინა მაგალითში გვქონდა:

> ერთი User-ს ჰყავს ერთი Profile.

აი აქ იგივე ხდება —
მაგრამ ახლა დამატებულია `{ cascade: true }`.

---

### მოდი გავაკეთოთ ანალოგია 🎒

წარმოიდგინე, შენ ხარ **User**, და გაქვს შენი **Profile**.

ახლა წარმოიდგინე, რომ შენს პროფილს შენ თვითონ ქმნი:

> როცა შენ გაჩნდები (User შეიქმნება),
> ავტომატურად შეიქმნება შენი პროფილიც (Profile).

---

ამას ზუსტად აკეთებს **`cascade: true`**

ანუ ნიშნავს:

> "თუ მე (User) რაღაცას გავაკეთებ —
> მაგალითად, შევიქმნი, შევცვლი ან წავშლი —
> იგივე ავტომატურად მოხდეს ჩემს პროფილზეც (MetaOption-ზე)."

---

### 📘 მარტივი მაგალითი კოდით:

```ts
@OneToOne(() => MetaOption, { cascade: true })
@JoinColumn()
metaOption: MetaOption;
```

ახლა, როცა შენ შექმნი ახალ `User`-ს ასე:

```ts
const newUser = {
  name: 'Luka',
  metaOption: {
    theme: 'dark',
    language: 'ka',
  },
};

await userRepository.save(newUser);
```

👉 TypeORM თვითონ შეინახავს **ორივე ცხრილში** —
`User`-საც და `MetaOption`-საც.

---

### 🎯 მოკლედ რომ ვთქვათ

| მოქმედება    | ხდება `cascade: true`-ს გამო     |
| ------------ | -------------------------------- |
| User-ს ვქმნი | ავტომატურად შეიქმნება MetaOption |
| User-ს ვცვლი | შეიცვლება MetaOption             |
| User-ს ვშლი  | წაიშლება MetaOption              |

---

### 🧠 მარტივი შედარება:

წარმოიდგინე, რომ User და მისი MetaOption არიან **მშობელი და შვილი** 👨‍👦
თუ დედამიწიდან გაქრება მშობელი (User),
👉 ავტომატურად ქრება შვილიც (MetaOption)
ესაა “კასკადური” ქმედება — **cascade**.
