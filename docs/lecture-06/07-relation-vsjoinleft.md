## 🚀 1. `relations` — მარტივი გზა (TypeORM თვითონ აკეთებს JOIN-ს)

ეს გზა უფრო **სუფთა და ავტომატურია**.
შენ უბრალოდ ეუბნები TypeORM-ს:

> “გადამიბი ეს ცხრილიც.”

მაგალითად:

```ts
const posts = await postsRepository.find({
  relations: { metaOptions: true },
});
```

👉 TypeORM შიგნით ქმნის რაღაც ასეთ SQL-ს:

```sql
SELECT * FROM posts
LEFT JOIN meta_options ON posts.metaOptionsId = meta_options.id;
```

და აბრუნებს შედეგს **გაერთიანებულად**, ობიექტად:

```js
[
  {
    id: 1,
    title: 'Post 1',
    metaOptions: { id: 10, keywords: 'nestjs', description: '...' },
  },
];
```

🟢 მარტივი გამოსაყენებელია
🔴 მაგრამ ცოტა შეზღუდულია — მაგალითად, ვერ აყენებ `WHERE` მხოლოდ relation-ზე.

---

## ⚙️ 2. `leftJoinAndSelect` — უფრო ზუსტი, “ხელით” კონტროლი

ეს გამოიყენება **QueryBuilder-ში**, როცა გინდა მეტი კონტროლი:

```ts
const posts = await postsRepository
  .createQueryBuilder('post')
  .leftJoinAndSelect('post.metaOptions', 'meta')
  .where('meta.keywords LIKE :kw', { kw: '%nestjs%' })
  .orderBy('post.id', 'DESC')
  .getMany();
```

აქ შენ თვითონ ქმნი SQL-ის მსგავს კოდს:

- თავად წყვეტ რომელი ცხრილი როგორ შეაერთოს,
- შეგიძლია დაადო `WHERE`, `ORDER BY`, `LIMIT`, და ა.შ.

🟢 ძალიან მოქნილია
🔴 მაგრამ ცოტა მეტია დასაწერი და გაგება რთულდება დამწყებისთვის.

---

## 🧠 მარტივი შედარება ცხრილით

| თვისება                                          | `relations`           | `leftJoinAndSelect`      |
| ------------------------------------------------ | --------------------- | ------------------------ |
| გამოყენება                                       | მარტივი, მოკლე        | უფრო ზუსტი, მოქნილი      |
| JOIN ტიპი                                        | ავტომატურად LEFT JOIN | შენ წყვეტ, რა JOIN გინდა |
| შეგიძლია ფილტრი (`where`) relation-ზე?           | ❌ არა                | ✅ კი                    |
| შეგიძლია აირჩიო მხოლოდ კონკრეტული ველები?        | ❌ არა                | ✅ კი                    |
| უკეთესია API-სთვის, როცა უბრალოდ გინდა გამოყვანა | ✅                    | ⚪                       |
| უკეთესია რთული ძიებების/ფილტრებისთვის            | ⚪                    | ✅                       |

---

## 🎯 მარტივი ანალოგია:

| რეალური მაგალითი    | რას აკეთებს                                                                                   |
| ------------------- | --------------------------------------------------------------------------------------------- |
| `relations`         | ეუბნები მეგობარს: “მომიყვანე ყველა ბავშვი თავისი პროფილით.” — ის თვითონ მოიყვანს ყველაფერს.   |
| `leftJoinAndSelect` | შენ თვითონ მიდიხარ, არჩევ ვის მოიყვან, რა წესით, რა თანმიმდევრობით. — უფრო ზუსტად აკონტროლებ. |

---

👉 მოკლედ:

- როცა გინდა უბრალოდ გამოყვანა (`find`, `findOne`) — გამოიყენე **relations**
- როცა გჭირდება ფილტრები, JOIN-ზე პირობები ან რთული query — გამოიყენე **QueryBuilder + leftJoinAndSelect**
