## 🎯 მარტივი ახსნა:

`eager: true` ნიშნავს
👉 “ამ კავშირს ყოველთვის მოიტანე ავტომატურად, როცა ამ entity-ს ვიძახებ.”

---

## 💡 მაგალითი:

```ts
@ManyToOne(() => User, (user) => user.posts, {
  eager: true,
})
author: User;
```

ეს ნიშნავს:

> როცა მე გამოვიძახებ `Post`-ს, TypeORM ავტომატურად მომიტანს მის `author`-საც.
> ანუ აღარ მომიწევს ხელით მითითება `relations: ["author"]`.

---

### 🔍 მაგალითად, ასეთი query:

```ts
const post = await postRepository.findOne({
  where: { id: 1 },
});
```

თუ `eager: true` გაქვს:

```js
{
  id: 1,
  title: "My Post",
  author: { id: 5, name: "Mindia" } // 👈 ავტომატურად მოიტანა
}
```

თუ **არ გაქვს eager: true**:

```js
{
  id: 1,
  title: "My Post"
  // author არ ჩანს 👇
}
```

თუ გინდა რომ გამოჩნდეს, უნდა მიუთითო:

```ts
await postRepository.findOne({
  where: { id: 1 },
  relations: ['author'], // 👈 ხელით უნდა უთხრა
});
```

---

## 🧠 ანუ მოკლედ:

| პარამეტრი                       | რას აკეთებს                                                 |
| ------------------------------- | ----------------------------------------------------------- |
| `eager: false` _(ნაგულისხმევი)_ | კავშირი არ იტვირთება ავტომატურად — უნდა მიუთითო `relations` |
| `eager: true`                   | TypeORM ყოველთვის მოიტანს ამ კავშირს ავტომატურად            |

---

## ⚠️ გაფრთხილება:

`eager: true` კარგია მცირე და მარტივ პროექტებში,
მაგრამ **საყურადღებოა დიდ ბაზებზე**, რადგან:

- ყოველთვის მოიტანს extra join-ს
- ვერ გამორთავ კონკრეტულ query-ზე

ანუ შეიძლება **დაუფიქრებლად გადატვირთოს query-ები**,
თუ entity-ს ბევრი eager კავშირი აქვს (მაგ. author → posts → comments → user → ... და ა.შ. 😅)

---

## ✅ დასკვნა:

> `eager: true` = “მოიტანე ეს კავშირი ყოველთვის ავტომატურად”

🧩 იდეალურია, როცა მაგალითად `Post` ყოველთვის უნდა მოვიდეს თავის `author`-თან ერთად.
